## 1. 온보딩 플로우 전체 구조

```
[입국 전] → [입국/ARC 신청] → [휴대폰 개통] → [은행 계좌 개설] → [계좌 활성화/설정] → [일상 금융] → [고급 금융]
   S0          S1                 S2              S3                  S4                S5          S6
```

### 핵심 발견: "닭-달걀" 순환 의존성

```
┌─────────────────────────────────────────────────────────┐
│              순환 의존성 (Circular Dependency)              │
│                                                         │
│   ARC ←──────── 은행 계좌 개설에 필요                      │
│    │                    ↑                                │
│    ↓                    │                                │
│   휴대폰 개통 ────→ 은행 계좌 ←── 후불 폰 개통에 필요       │
│    ↑                    │                                │
│    │                    ↓                                │
│   선불폰(임시) ←── 인터넷뱅킹 활성화에 필요                 │
│                                                         │
│   해결 순서: ARC → 선불폰 → 계좌개설 → 후불폰 전환         │
│   BUT: 이 순서를 아는 유학생이 거의 없음                    │
└─────────────────────────────────────────────────────────┘
```

## 2. 단계별 상세 분석

### S0. 입국 전 (Pre-Arrival)
**해야 할 일:** 필요 서류 준비, 학교별 요구사항 파악
**현실:** 대부분 아무 준비 없이 입국

| 페인포인트 | 근거 | 심각도 |
|-----------|------|--------|
| 무엇을 준비해야 하는지 모름 | Reddit: "Moving to Seoul for work... first time in Korea" (17점, 17댓글) | ★★★ |
| 학교별 요구사항이 다름 | 연세대: 교내 우리은행 필수, 서울대: 우리/신한/NH 선택 | ★★★ |
| 정보가 흩어져 있음 | 학교, 은행, 이민청, 통신사 각각 별도 안내 | ★★★ |

### S1. ARC (외국인등록증) 취득
**해야 할 일:** 출입국관리사무소 방문, ARC 신청, 2~4주 대기
**핵심 변수:** 비자 유형 (D-2 유학, D-4 어학연수, E-2 교사, F-4 재외동포 등)

| 페인포인트 | 근거 | 심각도 |
|-----------|------|--------|
| ARC 발급까지 2~4주 대기, 그 사이 금융 접근 불가 | 웹 리서치: 표준 처리기간 3~4주 | ★★★★ |
| 비자 변경 시 ARC 번호 변경 → 은행 정보 업데이트 필요 | Reddit: "H-1→학생비자 변경 시 계좌는?" (0점, 6댓글) | ★★ |
| 디지털 ARC 수용 여부 은행마다 다름 | 2025.03.21부터 하나/신한/iM/부산/전북/제주만 수용 | ★★★ |
| ARC 없이 계좌개설 가능 여부 불명확 | SNU SISA: "일부 은행은 ARC 전 보조서류로 가능" | ★★ |

---

### S2. 휴대폰 개통
**해야 할 일:** 한국 전화번호 확보 (선불 → 후불 전환)
**핵심 변수:** 선불/후불, 통신사, 알뜰폰 여부

| 페인포인트 | 근거 | 심각도 |
|-----------|------|--------|
| "폰 개통하려면 계좌가 필요하다" vs "계좌 만들려면 폰이 필요하다" | Reddit: 13점, 19댓글 - 가장 명확한 닭달걀 사례 | ★★★★★ |
| 선불폰으로는 은행 본인인증 안 되는 경우 | Enkostay: "Don't use prepaid SIMs for online services" | ★★★★ |
| 알뜰폰 vs 대형통신사 차이를 모름 | Reddit 다수 포스트에서 알뜰폰 관련 혼란 | ★★★ |
| ARC 명의 전환 시점 혼란 | 선불 → 후불 전환 시 필요 서류 불명확 | ★★ |

## 3. 핵심 인사이트 요약

### 3.1 가장 심각한 5대 페인포인트 (데이터 기반 우선순위)

1. **닭-달걀 문제** (S2↔S3) - 폰번호와 계좌의 순환 의존. 해결 순서를 아는 유학생이 거의 없음
2. **지점별 대응 불일치** (S3) - 같은 은행인데 지점마다 다른 답변. 가장 높은 engagement
3. **이체한도 제한** (S5) - 일 100만원, 올려달라고 해도 거절. 전세금/등록금 이체 불가
4. **서류 불확실성** (S3) - 무슨 서류를 가져가야 하는지 공식 안내 없음, 재방문 반복
5. **정보 사일로** (S0~S6 전체) - 학교/은행/통신사/이민청 정보가 각각 분리, 통합 흐름 없음

### 3.2 유학생 특화 문제 (일반 외국인과 다른 점)

| 차이점 | 설명 |
|--------|------|
| 학교별 지정은행 | 장학금/등록금 관련 특정 은행 필수 (연세대→우리, 고려대→국민 등) |
| 비자 유형 제약 | D-2 비자: 사업자 등록 불가, 일부 금융상품 이용 제한 |
| 한국어 능력 편차 큼 | 한국어 초급~중급이 대다수, 금융 전문용어는 모름 |
| 학기 주기 정착 | 매 학기 신규 유학생 유입 → 같은 문제가 반복적으로 발생 |
| 부모 송금 의존도 높음 | 해외송금 수령이 핵심 니즈, 일반 근로자와 다름 |

### 3.3 기존 해결책의 한계 (Status Quo)

| 현재 해결책 | 한계 |
|------------|------|
| Reddit/커뮤니티 | 사후적 (막힌 뒤에 질문), 정보 파편적, 정확성 보장 없음 |
| 학교 국제처 | 학교마다 다름, 금융 전문성 부족, 은행별 차이 반영 못함 |
| 은행 FAQ/콜센터 | "질문을 알아야" 시작, 유학생 맥락 반영 안 됨 |
| 블로그/가이드 | 일반적 정보, 개인 조건(학교/비자/국적) 미반영 |
| Hana EZ 앱 | 기능은 있으나 "언제/왜/어떤 순서로" 연결 없음 |

## 4. 제안서 반영 포인트

### 데이터 기반 주장 (실제로 방어 가능한 수치)

- Reddit r/Living_in_Korea 1년간 금융 관련 261개 포스트 중 **43%(112건)가 온보딩 관련**
- 온보딩 관련 포스트의 **평균 댓글 수 12.3개** → 같은 문제를 겪는 사람이 많다는 증거
- **지점 불일치 문제** 관련 포스트 평균 engagement(score+댓글) **24.1** → 높은 공감
- 가장 engagement 높은 온보딩 포스트: "Is there any reason Korea is so secure?" (105점, 47댓글) → 인증/보안 절차 복잡성
- "닭-달걀" 포스트(13점, 19댓글): 댓글 수 대비 질문 빈도가 높음 → 검색으로 해결 안 되는 문제

### 제안서에서 수정이 필요한 부분

1. **설문 규모**: 실제 20명 → "20명 대상 파일럿 설문 및 온라인 커뮤니티 261건 정성 분석"으로 정직하게 표현
2. **문제 정의 구체화**: "금융 절차를 모른다"(모호) → "ARC-폰-계좌 순환의존성 + 지점별 불일치"(구체)
3. **데이터 출처 명시**: Reddit r/Living_in_Korea, Nowak & Partner, Study in Korea 통계 등
