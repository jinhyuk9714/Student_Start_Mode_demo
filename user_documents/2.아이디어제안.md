- **아이디어**
    - **유학생의 기본정보로 현재 상태를 진단해 맞춤 체크리스트를 생성하고, 단계별 AI 비서가 실행을 안내하고 문제를 해결하는 온보딩 서비스**
- **핵심 기능 - [입력→생성→대화→진행]**
    1. 온보딩 설문:  학교/과정/입국시기/비자·ARC/휴대폰/계좌 여부 등 최소 입력
    2. AI 체크리스트 자동 생성(메인) : 지금 가능 / 다음 / 잠김(아직 불가)로 정렬 
        - 체크리스트 내 버튼에서 그 해당 체크리스트 항목에 대한 이야기의 ai 채팅방으로 넘어감
        1. 단계별 AI 비서(보조): 외국인이 진행하면 생기는 의문 해결 / 
        2. 항목의 새 소식에 대해 선제적으로 채팅하기도 함
    3. 상태 변화 시 체크리스트 자동 업데이트 (예: “ARC 받았어요” → 잠김 해제 + 다음 단계 제시)
    4. 딥링크/웹링크 연결: Hana EZ 기능, HiKorea, 학교 포털 등 “바로 실행”으로 연결
    
    User Flow
    
- **차별점/장점**
    - 정보를 모아주는 서비스”가 아니라 “필요한 시점에 먼저 알려주는 구조” (pull→push 전환)
    - 기능 추가가 아니라 Hana EZ의 기존 기능을 ‘유학생 여정’으로 재연결 → 개발 리소스 대비 효과(전환율) 극대화
    - 체크리스트는 유학 DB + 룰 기반으로 신뢰성과 실현 가능성을 확보하고, AI는 보조(맥락 Q&A/상태 변화 감지)에 집중 → 환각/정책 리스크를 낮춘 설계
    - 학교상태별 분기(예: 지정 지점, 서류, 소요기간 등)를 반영해 “유학생 특화”를 명확히 함
- **구현 계획**
    - Phase 1: MVP(초기 검증)
        - 온보딩 설문 + 상태기반 체크리스트 + 단계별 AI 비서(보조) + 딥링크 연결
        - 상위 주요 학교/케이스부터(예: 3개교 수준) 룰/콘텐츠 구축, 다국어(한/영/중/베) 우선
    - Phase 2: 고도화
        - 학교 DB 확장, 체크리스트 정교화(잠김 조건/우선순위/알림), AI 비서에 학교별 FAQ·공식자료 기반 RAG 적용
    - Phase 3: 확장
        - 하나은행 데이터 활용
        - 동일 엔진으로 Worker Mode 등 외국인 타 세그먼트 확장(체크리스트만 교체)
- **예상 결과물**
    - 프로토타입 UI: 온보딩 설문 → 체크리스트(지금/다음/잠김) → 카드 상세/딥링크 → AI 비서 화면 흐름
    - 체크리스트 룰/콘텐츠 유학 DB(학교×상태 기반) + 상태 전이(ARC/계좌/번호 등) 정의서
    - “대표 유학생 2~3명” 시나리오별 데모(예: ARC 전/후, 번호 없음, 장학금 계좌 필요 등)
    - 검증 지표: 체크리스트 완료율, 막힘 구간, 재방문/문의 감소 체감, Hana EZ 기능 진입(딥링크) 전환률(데모 기준)
