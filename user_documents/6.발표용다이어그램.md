## 다이어그램 4: State Machine + Rule Engine (핵심 로직)

```
╔══════════════════════════════════════════════════════════════════╗
║           체크리스트 상태 머신  (State Machine)                    ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║                    ┌──────────┐                                  ║
║           ┌──────→│  LOCKED   │◄──────────────────────┐         ║
║           │       │  (잠김)   │   prerequisites       │         ║
║           │       └─────┬────┘   invalidated          │         ║
║           │             │                              │         ║
║           │             │ 선행조건                      │         ║
║           │             │ 모두 충족                     │         ║
║           │             ▼                              │         ║
║           │       ┌──────────┐                         │         ║
║           │       │ AVAILABLE│                         │         ║
║           │       │  (가능)  │                         │         ║
║           │       └─────┬────┘                         │         ║
║           │             │                              │         ║
║           │             │ 사용자 "시작"                  │         ║
║           │             ▼                              │         ║
║           │       ┌──────────┐                         │         ║
║           │       │IN_PROGRESS                         │         ║
║           │       │ (진행중) │                         │         ║
║           │       └─────┬────┘                         │         ║
║           │             │                              │         ║
║           │             │ 확인 (버튼 or AI 확인카드)     │         ║
║           │             ▼                              │         ║
║           │       ┌──────────┐     다른 항목의          │         ║
║           │       │COMPLETED │     잠김 해제            │         ║
║           │       │  (완료)  │────────────────────────→┘         ║
║           │       └──────────┘     (cascade)                     ║
║           │                                                      ║
║           │       ┌──────────┐                                   ║
║           │       │ REMOVED  │  ← 조건 변경 시 (비자 변경 등)      ║
║           │       │ (제거됨) │  → 조건 복구 시 LOCKED로 복귀       ║
║           │       └──────────┘                                   ║
║           │                                                      ║
║           └── ✗ 건너뛰기 불가 (LOCKED → COMPLETED)               ║
╚══════════════════════════════════════════════════════════════════╝


╔══════════════════════════════════════════════════════════════════╗
║              룰 엔진 실행 흐름  (5단계)                            ║
╠══════════════════════════════════════════════════════════════════╣
║                                                                  ║
║  입력: 학교 + 비자 + 현재 상태                                    ║
║                                                                  ║
║  ┌────────────────────────────────────────────────────────────┐  ║
║  │ STEP 1  조건부 체크리스트 항목 로드 (조건 매칭 필터링)        │  ║
║  │         required_condition 매칭 → 항목 동적 포함/제외        │  ║
║  └────────────────────────┬───────────────────────────────────┘  ║
║                           ▼                                      ║
║  ┌────────────────────────────────────────────────────────────┐  ║
║  │ STEP 2  학교별 오버라이드 적용 (조건부 분기 포함)             │  ║
║  │         예: 연세대 + 장학금 → 우리은행 항목 추가              │  ║
║  └────────────────────────┬───────────────────────────────────┘  ║
║                           ▼                                      ║
║  ┌────────────────────────────────────────────────────────────┐  ║
║  │ STEP 3  비자별 은행 요건 필터                                │  ║
║  │         예: D-2 + 하나은행 → 재학증명서 필요                  │  ║
║  └────────────────────────┬───────────────────────────────────┘  ║
║                           ▼                                      ║
║  ┌────────────────────────────────────────────────────────────┐  ║
║  │ STEP 4  선행조건 평가 → 상태 결정 (핵심)                     │  ║
║  │         각 항목의 prerequisite를 user_conditions와 비교       │  ║
║  │         → AVAILABLE / LOCKED / COMPLETED                    │  ║
║  └────────────────────────┬───────────────────────────────────┘  ║
║                           ▼                                      ║
║  ┌────────────────────────────────────────────────────────────┐  ║
║  │ STEP 5  정렬 & 반환                                        │  ║
║  │         지금 가능 → 다음 단계(잠김) → 완료됨                  │  ║
║  └────────────────────────────────────────────────────────────┘  ║
║                                                                  ║
║  출력: [{ 항목, 상태, 막힌이유, 예상소요, 딥링크 }]               ║
╚══════════════════════════════════════════════════════════════════╝
```
